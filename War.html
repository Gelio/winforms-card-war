Windows Forms

War - card game

The goal is to implement simplified version of War - card game (not described issues should imitate example application)

Part I

    • Main window - maximize button is inactive, resizing is not possible
    • Top of the window: menu bar consists of buttons "Game" and "About"
    • Button "Game" contains submenus "New Game" and "Exit"
    • Button "About" displays short info about the app
    • Button "New Game" - launches window, in which user can define:
      • User name
      • CPU name
      • Number of rounds to be played in a game
    • In new game window, the input data is verified (in case of incorrect or empty input, information about the error should be displayed)
    • Button "Exit" closes application
    • The rest of main window: "table"
      • There are 4 buttons and a few labels on the screen
      • Every text displayed depends on input data and current state of the game (line in the example)
      • Only the button representing player's cards generates next round
    • Game logic:
      • Every round, 2 numbers are randomly generated - for user and CPU opponent. Numbers are integers form 1 to 10
      • Higher number gives proper player a point
      • In case of draw noone gets a point
      • The button color, on which the higher number is displayed is changed to Color.Green, lower number - Color.Red, draw - Color.Orange
      • After playing number of rounds defined by user, a window pops up informing about game state and sugests another game
    • App can be closed:
      • By clicking 'X' in the main window
      • By clicking 'X' in new game window (only after running application - first new game window)
      • By clicking "Exit" in "Game" menu
      • By combination ALT+F4
      • After finishing the game (after infomation about game result, if user decides not to start another game)
    • Before closing the app, messagebox pops up to ensure that the user wants to close the app
    • App starts displaying new game window, main window is not visible
  • Tips:
    • DialogResult, MessageBoxButtons, MessageBoxIcon
    • Button, Label
    • FormClosing, FormClosingEventArgs.Cancel
    • FormBorderStyle
  • Scoring:
    • Main window and game logic: 4 points
    • Full implementation: 8 points

Part II

  • Only the main window is shown on the menu bar
  • New game window has a new field to fill - sets - defines number of sets of cards 1-10 in the game (value 1 means, there are 10 cards in the game, 2-20, etc)
  • Menu "Game" has 2 new buttons - Save and Load (files saved in own format, for example *.mygame)
  • Menu has new button "Highscores"
  • Main window:
    • User can scale the main window (and maximize). Defined (reasonable) min size, max size is not defined
    • Scaling window scales the controls
    • Buttons representing user's and CPU's hands display a picture (back.bmp)
    • On the bottom of the window, in a frame, there is a panel responsible for setting the auto gameplay
    • Trackbar defines speed of auto gameplay (from 1 round per second to 1 round per 10ms, trackbar step is 10 ms)
    • Textbox defines last round to be played by auto gameplay
    • Button changes text between "Start"/"Stop" depending, if the auto gameplay is truned on or off
  • Game logic:
    • The deck contains (sets * 10) cards (sets defined in new game window) - sets cards 1, sets cards 2, etc
    • At the beginning, every player gets the same number of random cards from the deck, all cards are given away
    • The points a player has in each round is based on the number of cards in "hand" of a player
    • If a player wins a round, the cards from table are added to his hand - to the end of a list
    • In case of a draw on a table, the war begins - next cards are put on table upside down, and the following ones are being compared
    • The buttons representing cards in game display pictures (file numerki.gif), or the back of a card in case of draw in previus round
    • File numerki.gif is created in a way that every number has the hight of a picture and is 100px wide
    • The game ends after playing number of rounds defined in new game window, or if any of players loses the last card
    • After the game ends, the user name, score and number of played rounds is saved to highscores
  • Auto gameplay:
    • Next round is generated every timer tick (interval defined by trackbar)
    • Auto gameplay is ended if we reach the round defined as last by user, or if the game is over
    • Auto gameplay can be stopped by clicking "Stop" button
    • The pace of auto gameplay can be changed during auto gameplay
  • After the game, app informs about the result and suggests showing postgame stats
  • Postgame stats window:
    • 2 grapsh are displayed each representing user's and CPU's points during the whole game
    • Window has to be closed to get back to main window
  • Highscores window:
    • Window displays results of played games
    • Highscores display name, points and number of played rounds
    • Data is sorted by points and number of rounds, like in example
    • Window has to be closed to get back to main window
  • Save/Load game - user can save and load game in each moment of the game. The app should assure, that after loading a saved game the state of table stays the same and there are proper cards in game. The postgame stats should show the same window if the game is played all at once and if the user decides to save and load the game

    • OpenFileDialog, SaveFileDialog
    • Timer.Interval, TrackBar.Value
    • Chart
